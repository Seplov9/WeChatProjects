## 页面路由与导航

| 页面路径 | tabbar | 说明 |
|----------|--------|------|
| pages/index/index | 是（tab 0） | "需求池"标签，展示所有活跃需求，支持搜索和 5 级筛选 |
| pages/post/post | 是（tab 1） | "发单"标签，发布新需求或编辑已有需求 |
| pages/profile/profile | 是（tab 2） | "我的"标签，微信登录 + 导航到我的发布/收藏/浏览记录 |
| pages/show/show | 否 | 需求详情页，展示完整需求信息和图片轮播 |
| pages/order-list/order-list | 否 | 统一列表页，根据 type 参数展示我的发布/收藏/浏览记录 |

- tabbar 采用 `custom: true` 自定义组件实现，代码在 `custom-tab-bar/`
- 每个 tab 页面在 `onShow` 中通过 `this.getTabBar().setData({ selected: N })` 设置选中态
- 非 tab 页面不会显示底部导航栏，通过顶部导航栏返回
- profile → order-list 使用 `wx.navigateTo`
- order-list（我的发布）→ post 编辑模式：使用 `wx.switchTab` + `app.globalData.pendingEditId` 传参
- index/order-list → show：使用 `wx.navigateTo` + 查询参数 `?orderId=xxx`

## 云函数 API（quickstartFunctions）

通过 `event.type` 字段区分操作，共 16 个类型：

| type | 功能 | 参数 | 涉及集合 |
|------|------|------|-----------|
| getOpenId | 获取用户 openid | 无 | - |
| getMiniProgramCode | 生成小程序码 | 无 | - |
| getPhoneNumber | 解密手机号（个体工商户后可用） | event.code | - |
| addUserData | 写入提交数据（含内容安全审核） | event.content | users（type: "submission"） |
| getUserData | 查询所有提交数据（按时间倒序） | 无 | users（type: "submission"） |
| saveUser | 保存/更新用户档案（含昵称安全审核） | event.nickName, event.avatarUrl | users（用户档案） |
| createOrder | 发布需求（含内容安全审核） | event.category, title, content, reward, contact, city, images, publisherName, publisherAvatar | orders |
| getOrders | 查询活跃需求池（status=active，按 boost+时间排序，限 50 条） | 无 | orders |
| getMyOrders | 查询当前用户发布的所有需求（按时间倒序） | 无 | orders |
| getOrderById | 查询单个需求详情 | event.orderId | orders |
| updateOrder | 更新需求（验证发布者身份 + 内容安全审核） | event.orderId, category, title, content, reward, contact, city, images | orders |
| cancelOrder | 撤销需求（验证发布者身份，设置 status=closed） | event.orderId | orders |
| toggleFavorite | 切换收藏状态（已收藏则删除，未收藏则新增） | event.orderId, orderSnapshot | favorites |
| getFavorites | 查询当前用户的收藏列表（按时间倒序） | 无 | favorites |
| recordHistory | 记录浏览历史（同一需求去重：先删后插） | event.orderId, orderSnapshot | history |
| getHistory | 查询当前用户的浏览记录（按时间倒序，限 50 条） | 无 | history |

### 内容安全审核

以下操作在写入前调用 `cloud.openapi.security.msgSecCheck` 审核文本内容：

| 函数 | 审核字段 | scene | 违规返回 |
|------|---------|-------|---------|
| saveUser | nickName | 1（用户资料） | `errMsg: "内容不合规，请修改"` |
| addUserData | inputContent | 2（用户生成内容） | `errMsg: "内容不合规，请修改"` |
| createOrder | title + " " + content | 2（用户生成内容） | `errMsg: "内容包含不合规信息，请修改后重新发布"` |
| updateOrder | title + " " + content | 2（用户生成内容） | `errMsg: "内容包含不合规信息，请修改后重新发布"` |

- `suggest === "pass"` 放行，其余均拦截
- API 调用异常（如个人主体无权限）时自动降级放行，不阻塞正常使用
- `config.json` 已声明 `security.msgSecCheck` 权限

## 数据库集合

### orders（需求表）

| 字段 | 类型 | 说明 |
|------|------|------|
| _id | string | 自动生成 |
| publisherId | string | 发布者 openid（云函数服务端取值） |
| publisherName | string | 发布者昵称 |
| publisherAvatar | string | 发布者头像 |
| category | string | 需求分类（如"家政保洁"、"游戏陪玩"等） |
| title | string | 需求标题（限 50 字） |
| content | string | 需求详情（限 500 字） |
| reward | number | 报酬金额 |
| contact | string | 联系方式（微信/电话） |
| city | string | 城市信息，格式为"省·市·区"（如"广东·深圳·南山区"），各级均可缺省 |
| images | array | 图片 cloud fileID 数组（可选，最多 9 张） |
| status | string | 状态："active"（进行中）或 "closed"（已结束） |
| boost | number | 推广权重，默认 0 |
| createdAt | date | 发布时间 |
| updatedAt | date | 最近更新时间 |

**设计要点**：
- `publisherId` 在云函数中从 `cloud.getWXContext().OPENID` 获取，无法伪造
- `city` 使用"·"分隔符拼接省市区，筛选时通过 `includes()` 子串匹配
- `images` 先通过 `wx.cloud.uploadFile` 上传至云存储，再将 fileID 数组写入文档
- `status` 为 "closed" 的需求不会出现在需求池（getOrders 过滤 `status: "active"`）
- getOrders 按 boost 降序后按 createdAt 降序排列，限制 50 条

### favorites（收藏表）

| 字段 | 类型 | 说明 |
|------|------|------|
| _id | string | 自动生成 |
| userId | string | 收藏者 openid |
| orderId | string | 被收藏需求的 _id |
| orderSnapshot | object | 收藏时需求的快照（category, title, content, reward, contact, city, images） |
| createdAt | date | 收藏时间 |

**设计要点**：
- 同一用户对同一需求只有一条记录（toggle 逻辑：存在则删除，不存在则新增）
- `orderSnapshot` 保存快照，即使原需求被修改或删除，收藏列表仍可展示当时的信息
- 收藏列表按 `createdAt` 倒序排列

### history（浏览记录表）

| 字段 | 类型 | 说明 |
|------|------|------|
| _id | string | 自动生成 |
| userId | string | 浏览者 openid |
| orderId | string | 被浏览需求的 _id |
| orderSnapshot | object | 浏览时需求的快照（category, title, content, reward, contact, city, images） |
| viewedAt | date | 浏览时间 |

**设计要点**：
- 同一用户浏览同一需求多次，只保留最新一条（去重逻辑：先删旧记录，再插入新记录）
- `orderSnapshot` 保存快照，与 favorites 设计一致
- 浏览列表按 `viewedAt` 倒序排列，限制 50 条

### users（用户表）

存放两类文档，通过字段区分：

**用户档案**（saveUser 写入）：

| 字段 | 类型 | 说明 |
|------|------|------|
| _id | string | 自动生成 |
| openid | string | 用户唯一标识（云函数服务端取值） |
| nickName | string | 微信昵称 |
| avatarUrl | string | 微信头像 |
| phoneNumber | string | 预留，当前为空，个体工商户后可填充 |
| memberType | string | 预留，会员类型 |
| memberExpire | date | 预留，会员到期时间 |
| createdAt | date | 首次登录时间 |
| updatedAt | date | 最近登录时间 |

**提交记录**（addUserData 写入，旧 demo 遗留）：

| 字段 | 类型 | 说明 |
|------|------|------|
| _id | string | 自动生成 |
| type | string | 固定值 "submission"，用于区分档案与提交 |
| userId | string | 用户 openid |
| inputTime | string | 提交时间（北京时间） |
| inputContent | string | 用户输入内容 |

**设计要点**：以 `openid` 为唯一标识，`phoneNumber`/`memberType`/`memberExpire` 已预留字段，升级个体工商户后无需改表。

## 页面功能详情

### 需求池（pages/index/index）

**布局**：
- 顶部搜索栏：白色圆角输入框（72rpx 高，36rpx 圆角），支持关键字搜索标题
- 两行筛选区（居中排列，各元素 flex-shrink: 0）：
  - 第一行：线上/线下（110rpx）+ 品类（200rpx），联动过滤
  - 第二行：省（160rpx）+ 市（160rpx）+ 区（160rpx），三级级联
- 每个 picker 样式统一：60rpx 高，白色背景，8rpx 圆角，24rpx 字号，文字溢出省略
- 卡片列表：占满页面宽度（border-radius: 0），卡片间 4rpx 灰色间隔

**交互逻辑**：
- 搜索：输入文字后回车触发筛选，按标题 `includes()` 匹配
- 线上/线下切换：重置品类为"全部品类"，重置省市区的所有选择，联动更新品类可选项
- 品类切换：仅当线上/线下不为"全部"时联动过滤品类选项；筛选时按具体分类名精确匹配
- 省切换：级联更新市列表和区列表（数据来自 `data/regions.js` 的 34 个省级行政区）
- 市切换：级联更新区列表
- 筛选逻辑：省/市/区均使用 `includes()` 对 city 字段做子串匹配
- 卡片展示：第一张图片（280rpx 高）+ 分类标签 + 报酬（红色 ¥）+ 收藏按钮 + 标题 + 内容（两行溢出）+ 城市 + 发布时间
- 收藏按钮：空心五角星 ☆（灰色 #ddd），已收藏为实心 ★（黄色 #f5a623），点击切换
- 点击卡片：记录浏览历史 → navigateTo 详情页
- onShow 时自动拉取需求列表和收藏状态

**关键样式参数**：
- 容器底部留白：`calc(100rpx + env(safe-area-inset-bottom) + 20rpx)`，防止 tabbar 遮挡
- 卡片内边距：28rpx 24rpx
- 卡片图片：宽 100%，高 280rpx，圆角 8rpx
- 收藏按钮：48rpx × 48rpx，字号 36rpx

### 发单（pages/post/post）

**布局**：
- 头部：📋 图标 + "发布需求"标题 + 说明文字（居中）
- 需求分类区（白色圆角 section，28rpx 内边距）：
  - 两行下拉筛选（与需求池样式统一），线上/线下 + 品类 + 省/市/区

  - 虚线边框的"+"添加按钮

  - 标题输入框（限 50 字，右下角字数统计）
  - 详情文本域（限 500 字，200rpx 高，右下角字数统计）
- 报酬与联系区（白色 section）：
  - 薪酬输入（¥ 前缀，数字键盘）+ 微信/电话输入，双列并排
- 提交区：
  - 新建模式：绿色渐变全宽按钮"立即发布" + 提示文字
  - 编辑模式：两个并排按钮 — 绿色"保存修改"（左）+ 红色描边"撤销发布"（右）

**交互逻辑**：
- 表单校验（强制的）：
  - 线上/线下不可为"全部" → 提示"请选择线上/线下"
  - 品类不可为"全部品类" → 提示"请选择品类"
  - 标题、内容、有效薪酬、联系方式均不可为空
- 城市信息可选填（省可以为"全部城市"）
- city 字段构造："省" + (如果选了市则 "·市") + (如果选了区则 "·区")，未选省则为空字符串
- 图片上传：`wx.chooseMedia`（压缩）→ `wx.cloud.uploadFile` 批量上传至 `orders/` 目录 → 存储 fileID
- 提交时调用 `createOrder` 或 `updateOrder`（取决于是否有 editId）
- 新建成功：清空所有表单字段，恢复默认值
- 编辑成功：保留当前编辑状态，可继续修改
- 撤销发布：弹出确认框 → 调用 `cancelOrder` 设置 status=closed → 清空表单和 editId

**编辑模式进入**：
1. 从 order-list（我的发布）点击卡片 → 设置 `app.globalData.pendingEditId` → `wx.switchTab` 到 post
2. post 的 `onShow` 检测 `pendingEditId` → 设置 editId → `loadOrder(id)` 拉取数据
3. `loadOrder` 解析 city 字段（"省·市·区" 格式）还原省市区选择器的索引状态
4. 表单回填所有字段（分类、标题、内容、薪酬、联系方式、图片、城市选择）

**关键样式参数**：
- 页面底部留白：`calc(100rpx + env(safe-area-inset-bottom) + 40rpx)`
- 提交按钮：96rpx 高，48rpx 圆角，渐变绿，阴影
- 撤销按钮：96rpx 高，48rpx 圆角，白色背景 + 红色边框 + 红色文字
- 两按钮并排：`display: flex; gap: 20rpx`，各占 `flex: 1`
- 文本域：200rpx 高
- 图片缩略图：144rpx × 144rpx，圆角 10rpx

### 统一列表页（pages/order-list/order-list）

接收 `?type=myOrders|favorites|history` 参数，动态切换数据源和标题。

**布局**：
- 顶部搜索栏：与需求池样式一致
- 两行筛选区：与需求池完全一致（5 个下拉框，级联逻辑相同）
- 卡片列表：占满页面宽度，与需求池样式一致

**交互逻辑**：
- 数据源切换：`pageType` 决定调用 `getMyOrders` / `getFavorites` / `getHistory`
- 筛选适配：数据形状不同时通过 `getCat()` / `getTitle()` / `getCity()` 辅助函数统一取值路径
  - myOrders 数据扁平：`item.category`、`item.title`、`item.city`
  - favorites/history 数据嵌套：`item.orderSnapshot.category`、`item.orderSnapshot.title`、`item.orderSnapshot.city`
- 分类筛选：使用 `this.data.categoryOptions[categoryIndex]` 取值（修复了早期用 `allCategoryOptions` 的 bug）
- 卡片点击：
  - 我的发布 → `wx.switchTab` 到 /pages/post/post（通过 `app.globalData.pendingEditId` 传 orderId）→ 进入编辑模式
  - 收藏/浏览记录 → `wx.navigateTo` 到 /pages/show/show?orderId=xxx → 进入详情页
- 我的发布卡片额外显示状态标签：绿色"进行中"（active）/ 灰色"已结束"（closed）

**关键样式参数**：

## 用户强调过的要求

### 登录方式
- 当前使用 `open-type="chooseAvatar"` 获取头像 + `<input type="nickname">` 获取昵称
- 旧的 `open-type="getUserInfo"` 已失效（只返回默认灰色头像和"微信用户"）
- `wx.getUserProfile` 已失效，不可用
- 手机号快捷登录（`open-type="getPhoneNumber"`）仅对企业/个体工商户开放，个人主体暂无权限

### 登录持久化
- 登录成功后通过 `wx.setStorageSync` 保存 `userInfo` 到本地
- `onLoad` 时通过 `wx.getStorageSync` 恢复，避免每次打开小程序都需重新登录
- 退出登录时 `wx.removeStorageSync` 清除

### 样式原则
- 不要随意修改已有的页面样式，除非明确指令
- 多个页面共用的 UI 组件（筛选区、卡片、搜索栏）样式必须统一
- 筛选下拉框统一样式：60rpx 高，白色背景，8rpx 圆角，24rpx 字号，文字溢出省略
- 按钮文字居中：使用 `display: flex; align-items: center; justify-content: center; padding: 0;`
- 卡片占满页面宽度：`border-radius: 0`，无阴影，卡片间用细缝分隔
- 弹窗需避开自定义 tabbar：`padding-bottom` 包含 `100rpx + env(safe-area-inset-bottom)`
- 弹窗层级需高于自定义 tabbar：mask `z-index: 9999`，popup `z-index: 10000`

### 筛选与导航
- 需求池和列表页的搜索+筛选功能完全一致，代码可复用
- 发单页表单校验：线上/线下和品类必须选择具体分类，城市可留空
- tab 页之间跳转用 `wx.switchTab`（不能传 query 参数，用 `app.globalData` 替代）
- 非 tab 页之间跳转用 `wx.navigateTo`
- 编辑模式：从我的发布进入发单页，表单回填原有数据

### 数据库设计
- 全部用户数据通过 `openid`（云函数服务端取值）关联，无法伪造
- 收藏和浏览记录保存 orderSnapshot 快照，防止原需求删除后数据丢失
- 城市信息（city 字段）以"省·市·区"格式存储，筛选用 `includes()` 子串匹配
- `phoneNumber`/`memberType`/`memberExpire` 字段已预留，便于升级个体工商户后直接填充

### 开发习惯
- 修改代码聚焦在用户真正提出的问题上，不要顺手改其他东西
- 同一个问题出现在多个页面时，一并修复，保持一致

### 云函数
- 本地调试时操作的是真实云端数据库，不是本地数据库
- 上传云函数推荐用"云端安装依赖（不上传 node_modules）"

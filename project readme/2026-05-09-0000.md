## 页面路由与导航

| 页面路径 | tabbar | 说明 |
|----------|--------|------|
| pages/index/index | 是（tab 0） | "数据"标签，数据输入 + 跳转到展示页 |
| pages/profile/profile | 是（tab 1） | "我的"标签，微信登录 + 用户信息展示 |
| pages/show/show | 否 | 数据列表展示，从 index 通过 navigateTo 进入 |

- tabbar 采用 `custom: true` 自定义组件方式实现，代码在 `custom-tab-bar/`
- 每个 tab 页面在 `onShow` 中通过 `this.getTabBar().setData({ selected: N })` 设置选中态
- 非 tab 页面不会显示底部导航栏

## 云函数 API（quickstartFunctions）

通过 `event.type` 字段区分操作：

| type | 功能 | 参数 | 涉及集合 |
|------|------|------|-----------|
| addUserData | 写入提交数据 | event.content | users（type: "submission"） |
| getUserData | 查询所有提交数据（按时间倒序） | 无 | users（type: "submission"） |
| saveUser | 保存/更新用户档案 | event.nickName, event.avatarUrl | users（用户档案） |
| getOpenId | 获取用户 openid | 无 | - |
| getPhoneNumber | 解密手机号（个体工商户后可用） | event.code | - |
| getMiniProgramCode | 生成小程序码 | 无 | - |

## 数据库集合

### users（唯一集合）

存放两类文档，通过字段区分：

**用户档案**（saveUser 写入）：

| 字段 | 类型 | 说明 |
|------|------|------|
| _id | string | 自动生成 |
| openid | string | 用户唯一标识（云函数服务端取值） |
| nickName | string | 微信昵称 |
| avatarUrl | string | 微信头像 |
| phoneNumber | string | 预留，当前为空，个体工商户后可填充 |
| createdAt | date | 首次登录时间 |
| updatedAt | date | 最近登录时间 |

**提交记录**（addUserData 写入）：

| 字段 | 类型 | 说明 |
|------|------|------|
| _id | string | 自动生成 |
| type | string | 固定值 "submission"，用于区分档案与提交 |
| userId | string | 用户 openid |
| inputTime | string | 提交时间（北京时间） |
| inputContent | string | 用户输入内容 |

**设计要点**：以 `openid` 为唯一索引，`phoneNumber` 预留空字段，升级个体工商户后无需改表。

## 页面功能详情

### 首页 - 数据输入（pages/index/index）

- 标题："数据"
- 输入框 + 绿色的"提交"按钮
- 提交调用 `addUserData`，写入 userId、inputTime、inputContent 到 users 集合（type: "submission"）
- 底部蓝色链接"查看已提交数据 →"，跳转到 show 页面

### 数据展示页（pages/show/show）

- 标题："数据列表"
- `onShow` 时自动调用 `getUserData` 拉取数据
- 卡片列表展示每条记录的 userId、inputTime、inputContent
- 无数据时显示"暂无数据"

### 个人中心（pages/profile/profile）

**未登录状态**：
- 顶部白底圆角 box（140rpx 高），左侧圆形灰色默认头像（80rpx）+ 右侧"请登录"文字
- 点击 box 弹出底部弹窗

**登录弹窗**（由下往上滑入动画）：
- 标题"欢迎登录"
- 头像选择区：使用 `<button open-type="chooseAvatar">`，点击唤起微信原生头像选择器，获取真实微信头像
- 昵称输入区：使用 `<input type="nickname">`，聚焦时微信键盘自动带出当前昵称，可编辑
- "请您同意"勾选框：必须在点击"完成登录"前勾选
- 未勾选时"完成登录"按钮为浅绿色禁用态，点击弹出"请先阅读并同意"
- 勾选后按钮变为亮绿色，点击后调用 `saveUser` 云函数写入/更新 users 集合

**登录后状态**：
- 左侧显示真实微信头像（圆形 80rpx）+ 右侧显示微信昵称
- box 下方显示红色"退出登录"按钮

**交互逻辑**：
- 使用 `open-type="chooseAvatar"` + `type="nickname"` 替代已失效的 `getUserInfo`
- 登录信息通过云函数 `saveUser` 以 openid 为唯一标识 upsert 写入

## 用户强调过的要求

### 登录方式
- 当前使用 `open-type="chooseAvatar"` 获取头像 + `<input type="nickname">` 获取昵称
- 旧的 `open-type="getUserInfo"` 已失效（只返回默认灰色头像和"微信用户"）
- `wx.getUserProfile` 已失效，不可用
- 手机号快捷登录（`open-type="getPhoneNumber"`）仅对企业/个体工商户开放，个人主体暂无权限

### UI 细节
- 按钮文字居中：使用 `display: flex; align-items: center; justify-content: center; padding: 0;`
- 底部弹窗需避开自定义 tabbar：`padding-bottom: calc(48rpx + 100rpx + env(safe-area-inset-bottom))`
- 弹窗层级需高于自定义 tabbar：mask `z-index: 9999`，popup `z-index: 10000`

### 数据库设计
- 全部数据统一存储在 `users` 集合
- 以 `openid`（云函数服务端取值）为唯一标识，无法伪造
- 用户档案通过 `nickName`/`avatarUrl` 字段识别，提交记录通过 `type: "submission"` 识别
- `phoneNumber` 字段已预留，便于升级个体工商户后直接填充

### 开发习惯
- 不要随意修改已有的页面样式，除非明确指出样式有问题
- 修改代码聚焦在用户真正提出的问题上，不要顺手改其他东西

### 云函数
- 本地调试时操作的是真实云端数据库，不是本地数据库
- 上传云函数推荐用"云端安装依赖（不上传 node_modules）"
